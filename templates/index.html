Naming Policy

The template defines how each client gets renamed in Omada. Tokens are replaced with the resolved value (AI-first, then Fingerbank, then Omada).

Quick presets:

Token Reference

{name}AI short_name → hostname → "Device"
{os}AI normalized_os → Omada osName → "Unknown"
{type}AI normalized_type → Omada deviceType → "Unknown"
{vdr}AI manufacturer → Omada vendor → "Unknown-NIC"
{host}DHCP hostname (raw)
{cat}AI type → Omada category → "Others"
{mac}MAC address (uppercase dash format)
{ip}IPv4 address
{notes}Owner notes (raw text)
